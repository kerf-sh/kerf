# IPC-D-356A Netlist Export

Two LLM tools generate a bare-board electrical test (BET) netlist and run
connectivity analysis on a CircuitJSON board.

Spec: **IPC-D-356A (1997)** — the standard format accepted by fab-house flying-
probe and bed-of-nails test systems for bare-board electrical test.

---

## `export_ipc_netlist`

Converts a CircuitJSON board into an **IPC-D-356A** netlist text file (`.IPC`).
Fab houses load this file into their EBT (electrical bare-board test) system to
verify that every net is continuous and that no unintended shorts exist between
nets.

**When to use:** The user asks to export a BET netlist, create an IPC-D-356
file, prepare for bare-board electrical test, or upload a netlist to a fab
house's test system.  Run `netlist_report` first to confirm there are no open
nets before submitting.

**Input:**
- `circuit_json` (required) — parsed CircuitJSON array from the active board.
- `stem` (optional) — job/board name used in the header and filename (default `"board"`).

**Output:**
- `filename` — e.g. `board.IPC`
- `content_b64` — base64-encoded netlist text; decode and save as `<stem>.IPC`.
- `record_count` — total net records emitted.
- `records_317_smt` — count of type-317 records (SMT pads, no drill).
- `records_327_pth` — count of type-327 records (PTH pads + vias).
- `preview` — first 20 lines of the file as plain text.
- `message` — human-readable summary.

### IPC-D-356A record types

| Record | Meaning                             | CircuitJSON sources                     |
|--------|-------------------------------------|-----------------------------------------|
| `317`  | SMT pad (not drilled) — §4.2.1      | `pcb_smtpad`, `pcb_component_pad`       |
| `327`  | PTH pad or via (drilled) — §4.2.2   | `pcb_plated_pad`, `pcb_pad` (with drill), `pcb_via` |

### File structure (IPC-D-356A §4)

```
C  IPC-D-356A Netlist — generated by Kerf Electronics
P  JOB     <stem>
P  CODE    IPC-D-356A
P  DATE    <ISO 8601 timestamp>
P  UNITS   MM
317 <net>          <REFDES>-<PIN> <flags> <X> <Y> <W>X<H><layer>
327 <net>          <REFDES>-<PIN> D P     <X> <Y> <W>X<H>00
...
999
```

### Fixed-width record layout (IPC-D-356A §4.2.3)

| Columns | Field       | Content                                              |
|---------|-------------|------------------------------------------------------|
| 1-3     | Record type | `317` (SMT) or `327` (PTH/via)                       |
| 5-18    | Net name    | 14 chars, left-justified, space-padded               |
| 20-30   | REFDES-PIN  | 6-char refdes + `-` + 4-char pin number              |
| 32      | Drilled     | `D` = drilled, ` ` = SMT (not drilled)              |
| 33      | Plated      | `P` = plated, `U` = non-plated, ` ` = SMT           |
| 34-35   | Mid-net     | `M ` = intermediate pad on net; `  ` = last/only pad |
| 36-44   | X coord     | ±DDDDDDD (sign + 7 digits; metric mode: mm × 10000)  |
| 45-52   | Y coord     | same format                                          |
| 53-56   | X size      | pad width in same units (4+ digits)                  |
| 57      | `X`         | size separator                                       |
| 58-61   | Y size      | pad height                                           |
| 62-63   | Layer       | `00` through-hole/via, `01` top, `02` bottom         |

Coordinates are in **metric mode** (`P UNITS MM`) where 1 mm = 10 000 counts
(0.001 mm resolution), matching the output of KiCad 6 and Altium.

### Mid-net flag (§4.2.3)

Within a net with N pads, the first N−1 records carry `M ` in the mid-net
field; the last record carries `  `.  This allows test systems to associate all
records for a net without sorting by net name.

### Net name extraction

The tool resolves net names from CircuitJSON pad elements using (in priority order):

1. `pad.net_id` or `pad.net_name`
2. `pad.source_net_id`
3. Falls back to `"N/C"` (not connected)

Pads assigned `"N/C"` are emitted in the netlist (a real open) and are also
reported by `netlist_report` as unconnected.

## `netlist_report`

Analyses a CircuitJSON board for IPC-D-356A connectivity problems **before**
export.

**When to use:** Before calling `export_ipc_netlist`, or when the user asks to
check for floating nets, open connections, or unassigned pads.

**Input:** `circuit_json` (required).

**Output:**
- `status` — `"OK"` or `"ISSUES_FOUND"`
- `nets_total` — unique named nets (excluding `N/C`)
- `connected_nets` — nets with ≥ 2 pads/vias
- `open_nets` — list of `{net, pad_count}` for nets with < 2 points
- `single_pad_nets` — subset where `pad_count == 1` (definite open)
- `unconnected_pads` — count of pads with no net assignment (`N/C`)
- `total_pads_vias` — total pad + via elements found
- `message` — human-readable summary

### Interpreting results

| Condition             | Meaning                                      | Action                          |
|-----------------------|----------------------------------------------|---------------------------------|
| `single_pad_nets`     | Net has exactly 1 pad — a wiring omission    | Add missing connection or trace |
| `open_nets` (≥ 1 pad) | Net has 0 pads — defined but unreachable     | Add at least 2 pads to the net  |
| `unconnected_pads`    | Pads with no net assignment                  | Assign a net or mark intentional |

A clean board for fab submission should return `status = "OK"` with
`open_nets = []` and `unconnected_pads = 0`.

| Section         | Content                                                    |
|-----------------|------------------------------------------------------------|
| IPC-D-356A §4   | Overall file structure: header + net records + 999 end    |
| IPC-D-356A §4.1 | Header records: `C` comment, `P JOB`, `P CODE`, `P DATE`, `P UNITS` |
| IPC-D-356A §4.2 | Net record format (fixed-width 59+ char lines)             |
| IPC-D-356A §4.2.1 | Record type 317: SMD/surface-mount pads                  |
| IPC-D-356A §4.2.2 | Record type 327: through-hole pads and vias              |
| IPC-D-356A §4.2.3 | Field definitions: coordinates, layer codes, flags       |
| IPC-D-356A §4.4 | End-of-file record: `999`                                  |
