# v0.2 milestone — "Manufacture & share"

**Target ship**: ~6-8 weeks after v0.1.0 (mid-July 2026).

v0.1.0 covered breadth — every CAD discipline, every plugin, the
full hosted stack, the open-source release shape. v0.2 picks the
**most-demanded gaps** for users who actually want to manufacture
something with Kerf, plus the Workshop / sharing polish that came
out of v0.1 telemetry.

The cut is **10 items**: small enough to ship coherently, big enough
to be a real milestone. The theme is **manufacture and share** —
slicing, more CAM, gallery improvements, the planned Sketcher and
NURBS items that unblock real CAD workflows.

## v0.2 items

### Manufacturing

1. **Slicing: plane-section / cross-section** — CAD-side feature op.
   Wraps `BRepAlgoAPI_Section` (binding already probed in NURBS C1).
   New `.section` file kind storing the 2D outline; section-plane
   gumball in the viewport. Small lift; OCCT already does the work.
   Roadmap row exists. ~1 sonnet-day.

2. **Slicing: CNC layered slicing** — stacks plane-sections at fixed Z
   heights for layered milling / waterjet / laser-stack workflows.
   New `.cam.layered` file kind built on top of the cross-section op.
   ~3 sonnet-days.

3. **5-axis CAM T5: G-code emission for constant-tilt finishing** —
   takes CL points produced by T3 and emits G-code with linear A/B
   moves. LinuxCNC + Fanuc post-processors. ~2 sonnet-days.

4. **5-axis CAM T6: G-code for 3+2 indexed** — emits a single A/B
   rotate + 3-axis G-code body. Same post-processors. ~2 sonnet-days.

5. **5-axis CAM T7: tool DB schema + UI** — `.tool` file kind with
   shank/flute/ball geometry. UI panel in CAMView. Required for T5/T6
   to produce realistic tool-tip math. ~2 sonnet-days.

### NURBS Phase 4 — keep the bleed alive

6. **NURBS Phase 4 Capability 1 T4-T6: end-to-end surface booleans** —
   inspector entry + integration tests + worker performance pass on
   real surface inputs. Capability 1 isn't complete until users can
   `cut(surface_a, surface_b)` and see a trimmed result in the
   viewport without hitting the binding fallback path. ~4 sonnet-days.

7. **NURBS Phase 4 Capability 2 T1-T3: trim-by-curve** — given a face
   + 3D curve, project + split + keep one side. Plan exists. Highest
   immediate user-visible value after surface booleans. ~5
   sonnet-days.

### Sketcher gaps

8. **Sketcher: Bezier curves** — add `bezier` entity kind to `.sketch`.
   planegcs already supports control-polygon constraints. UI: draw via
   control-point clicks. Tangent / G1 / G2 between segments. Roadmap
   row exists (cubic B-spline already shipped in v0.1; Bezier is the
   net-new bit). ~2 sonnet-days.

9. **Sketcher: symmetry over arbitrary construction line** — extends
   axis-aligned symmetry (shipped) to mirror across any user-drawn
   construction line. Roadmap row exists. ~2 sonnet-days.

### Workshop + sharing polish (telemetry-driven)

10. **Workshop auto-thumbnail re-capture on file change** — currently
    thumbnails capture on save but only the active file's view feeds
    in. After a multi-file project changes, the primary thumbnail can
    drift from the showcase image the user wants. Add a "regenerate
    thumbnail" button + a setting to pin a specific gallery image as
    the primary. ~2 sonnet-days.

## What v0.2 explicitly does NOT include

- **NURBS Phase 4 C3 (matchSrf)** — high-value but high-risk
  (OCCT binding gaps in `BRepFill_Filling` continuity constraints).
  Defer to v0.3 after C1/C2 ship to learn the binding surface.
- **NURBS Phase 4 C4 (G3 continuity)** — structurally impossible in
  stock OCCT per the Phase 4 plan-doc; will ship as curvature-comb
  visualization only when product gates approve.
- **PLC structured text (`.plc.st`)** — defer to v0.3 once Workshop
  telemetry confirms how many electronics users would actually use it.
- **Quad remesher (Instant Meshes)** — wait until SubD prep
  + downstream FEM both have telemetry-confirmed demand.
- **Revit IFC import** — defer to v0.3; BIM persona is small share
  of v0.1 signups (per pre-launch beta).
- **3D-print G-code slicing** — AGPL-licensed deps (Cura / PrusaSlicer)
  + 10-15 sonnet-day lift. Worth doing eventually but not in v0.2.
- **Rust / Go / Lua SDKs** — already roadmap-rowed, wait for
  user demand.

## Estimate

~25 sonnet-days total for the 10 items. At 2 parallel agents that's
~12-13 calendar days of agent time; allow 3-4 weeks elapsed for
review + manual testing + the inevitable scope creep.

## How we'll know v0.2 is ready

- All 10 ✅ rows in ROADMAP.md.
- Released tagged `v0.2.0` with a CHANGELOG entry.
- Slicing + 5-axis end-to-end demo works: open a model → section it
  → CAM the section → export real G-code that a hobbyist mill would
  cut.
- Workshop multi-image gallery has > 100 published projects with > 1
  gallery image in production telemetry.
- Sketcher gap-closure (Bezier + arbitrary symmetry) wraps the
  FreeCAD-Sketcher parity loop.

## Out-of-band items that could still bump in mid-cycle

- Bug-fix patches on shipped v0.1 items (always allowed).
- Critical security disclosures (always allowed).
- High-value user requests that fit in a sonnet-day (case-by-case).
